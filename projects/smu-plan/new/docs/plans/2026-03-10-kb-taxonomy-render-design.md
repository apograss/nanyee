# 知识库分类树与文章展示设计

## 目标

- 把知识库从单层字符串分类升级为可维护的两级分类树。
- 母项只允许管理员维护，子项允许登录用户在既有母项下创建和编辑图标。
- 提升文章详情页的 Markdown 展示质量，同时保留右侧段落评论机制。

## 现状

- `Article.category` 目前只是一个自由文本字符串。
- 编辑页通过普通输入框写入分类，无法约束层级和图标。
- 侧边栏的图标映射是硬编码，无法后台维护。
- 文章正文目前只做 `marked -> sanitize -> innerHTML`，排版能力弱。
- 段落评论依赖 `[data-article-body]` 内的段落节点，不能被破坏。

## 方案

### 1. 分类数据模型

- 新增 `WikiCategory` 表，采用两级树结构。
- 顶级分类代表“母项”，只能由管理员创建、编辑、排序和删除。
- 子分类归属于某个母项，允许登录用户创建、编辑名称和图标。
- 文章绑定到子分类，同时保留 `Article.category` 作为兼容字段和展示冗余值。

### 2. 分类管理接口

- 新增公共读取接口，返回分类树和文章计数。
- 新增分类写接口：
  - 管理员可以维护母项和子项。
  - 普通用户只能在已有母项下新增或编辑子项。
- 保留审核外的 Wiki 共建风格，不引入额外审批流。

### 3. 管理后台

- 在后台增加独立的知识库分类管理页。
- 页面提供：
  - 母项列表
  - 子项树
  - 图标编辑
  - 顺序调整
  - 将社区创建的子项重新归类

### 4. 编辑页

- 编辑页改为“先选母项，再选子项”。
- 登录用户可以直接在当前母项下创建新子项，并设置图标。
- 当选中已有子项时，允许用户编辑子项名称与图标。
- 文章提交时写入 `categoryId`，并同步写入兼容字符串字段。

### 5. 文章浏览页

- 保留 `[data-article-body]` 容器与现有段落评论组件。
- 优化 Markdown 输出结构和页面样式：
  - 标题层级与间距
  - 引用块、列表、表格、代码块
  - 图片、分隔线、链接样式
  - 顶部分类信息与摘要区
- 不改成组件化 AST 渲染，避免影响现有评论锚点机制。

## 风险与控制

- 旧文章没有 `categoryId`：通过迁移脚本生成默认母项并把旧分类导入为子项。
- 子项被社区用户频繁修改：保留后台统一整理入口，不做权限过细拆分。
- Markdown 美化影响评论锚点：只增强渲染结果和样式，不移除段落节点。
